# 🧑‍🏫 SOFTY

> 교사-학부모 소통 분쟁 리스크 완화를 위한 NLP 기반 AI 메시지 검토·기록 서비스입니다.

## 📖 Overview

SOFTY는 교사와 학부모가 필요한 내용을 기록으로 남기며 주고받을 수 있는 비동기 소통 서비스입니다.  
응답에 대한 부담을 줄이고, 필요한 내용을 구조적으로 전달하고 확인할 수 있는 소통 경험을 제공합니다.

웹 서비스에서는 **교사용 웹**과 **관리자 웹**으로 구성되어 있습니다.  
교사는 학급을 개설하고 학부모와 연결해 소통 내용을 관리할 수 있고, 관리자는 서비스 운영 지표와 사용자 피드백, AI 모델을 관리할 수 있습니다.

## ✨ Main Features

### Teacher

- 카카오 로그인 및 교사 가입
  - 카카오 로그인 후 교사 정보를 입력해 가입할 수 있습니다.
  - 가입이 완료되면 학급 코드를 발급받아 학부모와 연결을 시작할 수 있습니다.
- 학부모 문의 수신함
  - 학부모가 보낸 문의를 목록으로 확인할 수 있습니다.
  - 문의 의도, 처리 상태, 마지막 메시지, 안읽은 메시지 수를 한눈에 파악할 수 있습니다.
- 문의 상세 확인 및 답변 작성
  - 학부모와의 대화 내용을 확인하고 답변을 작성할 수 있습니다.
  - 문의 의도와 처리 상태를 함께 확인하며 대화를 이어갈 수 있습니다.
- AI 소통 어시스턴트
  - 교사가 작성한 답변에 대해 분쟁 가능성을 분석합니다.
  - 표현이 오해를 부를 가능성이 있을 경우 더 부드러운 추천 답변을 제공합니다.
- 추천 답변 적용 및 수정 전송
  - AI가 제안한 추천 답변을 그대로 적용하거나 일부 수정한 뒤 전송할 수 있습니다.
  - 분석 결과가 적절했는지 피드백을 남길 수 있습니다.
- 증빙 리포트 생성
  - 학부모와의 대화 내역을 PDF 리포트로 생성하고 다운로드할 수 있습니다.
- 설정 및 학급 관리
  - 교사 프로필과 학급 정보를 확인할 수 있습니다.
  - 근무시간을 설정하고, 필요 시 학급을 변경할 수 있습니다.

### Admin

- 관리자 로그인
  - 관리자 계정으로 로그인해 서비스 운영 현황을 확인할 수 있습니다.
- 추천 문장 채택률 대시보드
  - AI 추천 답변이 실제로 얼마나 활용되었는지 확인할 수 있습니다.
  - 그대로 사용, 일부 수정 후 사용, 미사용 현황을 지표와 차트로 제공합니다.
- 리스크 탐지 현황 대시보드
  - 전체 메시지 중 분쟁 리스크가 감지된 비율과 탐지 건수를 확인할 수 있습니다.
  - 서비스 내 리스크 탐지 기능의 활용 현황을 한눈에 파악할 수 있습니다.
- PDF 리포트 현황 대시보드
  - 생성된 PDF 리포트 수와 교사별 생성 현황을 차트와 목록으로 제공합니다.
- 오류 검토
  - 교사 피드백을 바탕으로 AI 분석 결과를 검토할 수 있습니다.
  - 적절하지 않았던 분석 사례를 확인하고 개선에 참고할 수 있습니다.
- AI 모델 성능 확인
  - 현재 운영 중인 모델 정보와 성능 평가 결과를 확인할 수 있습니다.
  - Precision, Recall, F1-score 등 주요 지표를 통해 모델 상태를 모니터링할 수 있습니다.

## ⚙️ Getting Started

### Prerequisites

- Node.js 18 이상
- npm

### 1. Repository Clone

```bash
git clone [레포지토리 주소]
cd [프로젝트 폴더명]
```

### 2. Install Dependencies

```bash
npm install
```

### 3. Run Development Server

```bash
npm run dev
```

## 🗂️ Project Structure

```markdown
src/
├─ assets/ # 프로젝트 내부에서 사용하는 정적 리소스
├─ components/ # 재사용 UI 컴포넌트
│ └─ common/ # 공통 컴포넌트
├─ constants/ # 상수, 라벨 맵, enum 매핑, 정규식
├─ features/ # 도메인별 상태 관리, TanStack Query 훅, 캐시 키, 타입 정의
├─ hooks/ # 공통 커스텀 훅
├─ layouts/ # 페이지 공통 레이아웃
├─ pages/ # 페이지 단위 컴포넌트
├─ providers/ # QueryProvider, ThemeProvider 등 전역 Provider
├─ router/ # 라우트 정의 및 권한별 분기 처리
├─ services/ # API 호출 로직
│ └─ http/ # axios 인스턴스, 인터셉터 설정
├─ stores/ # 전역 상태 관리 스토어
├─ styles/ # 글로벌 스타일, 디자인 시스템
├─ utils/ # 범용 유틸 함수
├─ App.tsx
└─ main.tsx
```
